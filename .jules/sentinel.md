## 2024-05-24 - Configure Global Security Headers for Vercel Deployments
**Vulnerability:** Missing security headers (CSP, X-Frame-Options, etc.). By default, Vercel deployments do not automatically inject a comprehensive suite of security headers. This leaves applications susceptible to attacks like clickjacking, MIME-sniffing, and certain types of cross-site scripting (XSS).
**Learning:** For Astro projects deployed on Vercel, simply relying on the `@astrojs/vercel` adapter is not sufficient to enforce global HTTP security headers across static and server-rendered routes. The recommended approach to ensure these headers are applied consistently at the edge network level is to define them explicitly in a `vercel.json` configuration file at the project root.
**Prevention:** Always include a `vercel.json` file defining strict security headers (e.g., `X-Frame-Options`, `X-Content-Type-Options`, `Strict-Transport-Security`, `Referrer-Policy`) when setting up new projects or migrating existing ones to Vercel, ensuring defense-in-depth is established from the infrastructure layer.
