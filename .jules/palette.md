## 2024-03-19 - Accessibility of Icon-Only Links and Decorative Images in Astro Components
**Learning:** In Astro components mapping over data sets like `cv.json` (e.g., `SocialGrid`, `LinkGrid`), icon-only links lacked `aria-label`s, causing screen readers to read the raw URL instead. Additionally, decorative icons accompanying links or serving as link contents lacked `alt=""` and `aria-hidden="true"`, causing screen readers to unnecessarily announce image filenames or URLs.
**Action:** When creating or modifying Astro components that map over structured data to generate links or display icons, always ensure icon-only links have descriptive `aria-label` attributes (e.g., using `item.title`), and ensure purely decorative or redundant images are hidden from assistive technologies using `alt=""` and `aria-hidden="true"`.
